# Pandocster

Конвейер сборки одного итогового документа (Word, [PDF](#что-с-pdf), EPUB, Markdown, LaTeX) из исходников в Markdown. Структура документа повторяет структуру каталогов с исходными файлами. Идея взята из [статьи на Habr](https://habr.com/ru/articles/987982/), реализация упрощена под задачу «просто собрать документ со структурой по каталогам».

---

## Задачи

1. **Структура по каталогам** — один документ из многих `.md`; уровни заголовков (H1, H2, …) задаются глубиной файла в дереве каталогов.
2. **Перекрёстные ссылки** — ссылки между файлами в исходниках остаются рабочими в итоговом документе (Pandoc «из коробки» их ломает).
3. **Картинки** — корректное внедрение изображений при исходниках в разных директориях; относительные пути нормализуются, медиа копируются в нужное место.

---

## Как использовать

Одна из главных целей была "не телепите мне мозг, дайте простой инструмент, чтобы сел и поехал". По этому использовать просто:

1. Поставить на хост [Python](https://www.python.org/downloads/) и [Pandoc](https://pandoc.org/installing.html)
2. Поместить в каталог [./src/](./src/) контент в виде сгруппированных по папкам *.md файлов
3. Последовательно запустить всего две команды:
   1. `$ ./pandocster.py src/ build/` — он подготовит *.md для верстки
   2. `$ ./generate.sh` — в каталоге [./out/](./out/) появятся результаты верстки в четырех форматах

В результате в [./out/](./out/) соберется итоговый документ сразу в нескольих форматах.

## Что с PDF

Пандок не может в кириллицу и быстрого и простого способа это забороть я не нашел пока.
Поэтому PDF — можно, но делается он путем печати вёрда в pdf.

## Как устроено

Об этом можно почитать в [./src/](./src/) или еще лучше [указанным способом](#как-использовать) собрать содержимое [./src/](./src/), почитать результат в каталоге [./out/](./out/) и сравнить с исходником, чтобы понять в деталях.
