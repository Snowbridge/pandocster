# generate.sh
<!-- @header-offset:4 -->

Скрипт собирает один итоговый документ (например, DOCX) из всех Markdown-файлов в каталоге `build/` с помощью Pandoc.

## Что делает скрипт

1. **Поиск исходников**  
   Команда `find . -wholename "./build/*.md" | sort` находит все `.md` файлы непосредственно в каталоге `./build/` (один уровень вложенности) и передаёт их в Pandoc в отсортированном порядке.

2. **Сборка документа**  
   Pandoc вызывается с параметрами:
   - **Вход** — список найденных `.md` из `build/`.
   - **Выход** — `out/all.docx` (формат DOCX).
   - **Оглавление** — `--toc --toc-depth=4`.
   - **Lua-фильтры** (по порядку):
     - `filters/header_offset.lua` — смещение уровней заголовков;
     - `filters/link_anchors.lua` — подстановка якорей из комментариев `@anchor`.
   - **Ресурсы** — `--resource-path=src/assets` (поиск картинок и т.п. относительно этого пути).
   - **Метаданные** — `--metadata-file=metadata.yml` (если файл есть).

## Предполагаемый порядок работы

Сначала выполняется подготовка:

```text
python pandocster.py src build
```

Затем — сборка:

```text
./generate.sh
```

В результате в каталоге `out/` создаётся файл `all.docx` — единый документ с оглавлением и применёнными фильтрами.
