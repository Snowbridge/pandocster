# pandocster.py
<!-- @header-offset:4 -->

Скрипт подготавливает исходные Markdown-файлы к сборке единого документа: копирует каталог в `build/`, обрабатывает каждый `.md` файл и формирует общий файл reference-<span id="DeP6fzKMABZ">ссылок</span>.

## Запуск

```text
python pandocster.py <исходный_каталог> [целевой_каталог]
```

- **исходный_каталог** — каталог с исходными `.md` (обычно `src/`).
- **целевой_каталог** — необязательный; по умолчанию используется каталог `build/` рядом со скриптом.

## Что делает скрипт

1. **Подготовка каталога сборки**  
   Удаляет существующий целевой каталог (например, `build/`) и создаёт его заново.

2. **Копирование**  
   Рекурсивно копирует всё содержимое исходного каталога в целевой.

3. **Обработка каждого Markdown-файла**  
   Для каждого `.md` в целевом каталоге (включая вложенные):

   - **Уровень вложенности**  
     По относительному пути файла вычисляется «глубина» (уровень). Файлы `_index.md` считаются на уровень меньше (каталог не увеличивает глубину).

   - **Заголовок в начале файла**  
     В начало файла добавляется блок с:
     - комментарием `<!-- @header-offset: N -->` (N — вычисленный уровень) для фильтра `header_offset.lua`;
     - комментариями `<!-- @anchor="..." -->` с закодированным именем файла для фильтра `link_anchors.lua`.

   - **Reference-style ссылки**  
     Строки вида `[id]: url` извлекаются из тела документа. Для локальных файлов `url` заменяется на якорь `#<encoded_filename>`. Все такие строки удаляются из текста. Список обработанных reference-ссылок накапливается по всем файлам.

   - **Картинки**  
     В путях к изображениям в синтаксисе `![title](path)` обрезается всё до первого вхождения `assets/` включительно. Итоговый путь — от `assets/` и дальше (например, `../../assets/img/logo.png` → `img/logo.png`).

4. **Файл 999-reflinks.md**  
   В корне целевого каталога создаётся файл `999-reflinks.md`, в котором перечислены все собранные reference-style ссылки (без дубликатов, в отсортированном виде). Его подключают при сборке, чтобы определения ссылок были доступны во всём документе.

## Итог

После запуска в целевом каталоге лежит копия исходников с добавленными в каждый `.md` служебными комментариями и нормализованными путями к картинкам, плюс один общий файл с reference-ссылками. Дальше сборка выполняется скриптом `generate.sh` (Pandoc с нужными фильтрами).
