# Фильтр link_anchors.lua
<!-- @header-offset:4 -->

Lua-фильтр для Pandoc, который **превращает служебные HTML-комментарии с меткой `@anchor` в настоящие якоря** (элементы с `id`) в целевом формате. По этим якорям потом работают ссылки вида `#имя_файла`.

Например, если кликнуть на [вот эту ссылку](#8YJlGDiWIzWom) в сгенерированном документе, то окажешься в разделе, описывающем особенности файла `_index.md`.
Это способ делать ссылку не на раздел, а на конкретную строку. Рабочий способ, но, к сожалению, он нарушают [MD051]. Мне пока не удалось ничего дельного придумать, чтобы и ссылки на произвольную строку были возможны, и MD051 уважить.

## Как работает

1. **Поиск маркера**  
   Фильтр просматривает блоки сырого HTML и ищет строки вида:
   ```html
   <!-- @anchor="encoded_filename" -->
   ```
   В кавычках — уже закодированное для URL имя (например, имя файла с экранированием спецсимволов).

2. **Подстановка якоря**  
   Вместо такого комментария в документ вставляется:
   - для **LaTeX** — `\label{encoded_filename}`;
   - для остальных форматов (HTML, DOCX и т.д.) — пустой блок (например, `Div` со `Span`), у которого задан атрибут `id = encoded_filename`. В HTML это даёт элемент с `id="..."`, к которому можно привязать ссылку `#...`.

## Связь с pandocster.py

- **pandocster.py** в начало каждого `.md` добавляет комментарии `<!-- @anchor="..." -->`, где в кавычках — URL-encoded имя этого файла (без пути).
- Локальные reference-style ссылки на другие `.md` в исходниках скрипт заменяет на якорные ссылки вида `#encoded_filename`, то есть на имя целевого файла.
- При сборке **link_anchors.lua** подставляет в документ якоря с такими же `id`, поэтому ссылки `#encoded_filename` начинают указывать на нужные места в итоговом документе.

## Итог

**link_anchors.lua** даёт возможность ссылаться между разными исходными файлами по якорям: скрипт задаёт имена якорей и заменяет ссылки на файлы на `#имя`, а фильтр создаёт в выходном документе элементы с соответствующими `id` (или `\label` в LaTeX).
