# Фильтр header_offset.lua
<!-- @header-offset:4 -->

Lua-фильтр для Pandoc, который **смещает уровень всех заголовков** в документе на заданное значение. Используется, чтобы при сборке из нескольких файлов заголовки из разных разделов не конфликтовали по уровню (H1, H2, …).

## Зачем это нужно

Смещение уровня даёт возможность **понижать или повышать** уровень заголовков — для всего документа или для отдельно взятого раздела (если перед ним вставить свой маркер `@header-offset`). Так можно согласовать иерархию при сборке из многих файлов и решать точечные задачи вёрстки.

Например: в одном файле нужны заголовки, чтобы структурировать контент, но в оглавлении (TOC) этот подраздел хочется показывать одной строкой, без вложенных пунктов. В начало файла (или после заголовка раздела) добавляют `<!-- @header-offset: N -->`, где **N больше, чем значение `--toc-depth`** в вызове Pandoc. Тогда в исходнике раздел по-прежнему структурирован заголовками, в итоговом документе структура сохраняется, а в оглавлении заголовки этого подраздела не попадают — они оказываются глубже максимальной глубины TOC.

В частности, если посмотреть внимательно на документ, который генерируется из данного документа, то можно увидеть, что в каждом md-файле, описывающем фильтры, заголовки есть, но этих заголовков нет в оглавлении. Это как раз достигается при помощи `<!-- @header-offset:4 -->` во второй строке текущего файла и всех его сиблингов.

## Как работает

1. **Маркер в HTML-комментарии**  
   Фильтр ищет в *.md документе блоки сырого HTML вида:
   
   ```html
   <!-- @header-offset:N -->
   ```

   где `N` — целое число (например, 1, 2, 3). Как только такой комментарий встречается, значение `N` запоминается и действует для всех последующих заголовков до следующего такого маркера. Сам комментарий из вывода удаляется.

2. **Изменение уровня заголовков**  
   Для каждого элемента `Header` уровень заголовка пересчитывается по формуле:
   ```text
   новый_уровень = старый_уровень + current_offset - 1
   ```
   То есть при `current_offset = 1` уровень не меняется (1+1-1=1, 2+1-1=2). При `current_offset = 2` заголовок первого уровня становится вторым, второй — третьим и т.д.

## Откуда берётся маркер

Скрипт **pandocster.py** в начало каждого обрабатываемого `.md` файла добавляет блок с `<!-- @header-offset: N -->`, где `N` — вычисленная «глубина» файла (по пути и правилу для _index.md). Таким образом, файлы из подкаталогов автоматически получают больший сдвиг, и иерархия разделов в едином документе выстраивается корректно.

## Итог

**header_offset.lua** обеспечивает единую согласованную иерархию заголовков при сборке многих Markdown-файлов в один документ, опираясь на маркеры `@header-offset`, которые расставляет pandocster.py.
