# Фильтр newpage.lua
<!-- @header-offset:4 -->

Lua-фильтр для Pandoc, который управляет разрывами страниц в итоговом документе: перед каждым заголовком первого уровня и в местах явного маркера `<!-- @new-page -->`.

## 1. Заголовки первого уровня — с новой страницы

Для каждого заголовка уровня 1 фильтр добавляет разрыв страницы **перед** ним. Реализация зависит от формата вывода:

- **LaTeX / Beamer** — к заголовку добавляется класс `page-break-before` и атрибут `page-break="true"` (настройка разрыва страницы делается в шаблоне, при необходимости с пакетом `needspace`).
- **HTML / HTML5 / EPUB** — к заголовку добавляется стиль `page-break-before: always;`.
- **DOCX (OpenXML)** — к заголовку добавляется класс `page-break-before` и в начало содержимого заголовка вставляется сырой OpenXML разрыва страницы `<w:br w:type="page"/>`.
- **ODT** — добавляется класс `page-break-before`.
- **Остальные форматы** — добавляется класс `page-break-before` (разрыв можно настроить в шаблоне или стилях).

В результате каждый раздел, начинающийся с H1, в печатном/экспортируемом виде начинается с новой страницы.

## 2. Маркер <!-- @new-page -->

В тексте можно вставить комментарий:

```html
<!-- @new-page -->
```

Фильтр заменяет его на разрыв страницы **после** текущего места. Поддерживаются форматы:

- asciidoc, context, epub, html, html5, latex, beamer, ms, ooxml, odt, docx;
- для остальных подставляется HTML-блок с `page-break-before`.

Для чисто TeX-вывода (формат заканчивается на `tex`) маркер `@new-page` не обрабатывается (остаётся только поведение для H1).

## Итог

**newpage.lua** даёт два способа управлять разрывами страниц: автоматически — перед каждым заголовком первого уровня, и вручную — через комментарий `<!-- @new-page -->` в нужном месте исходного Markdown.
